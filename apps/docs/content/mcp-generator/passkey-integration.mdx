# PasskeyKit Integration

TypeScript MCP servers include full PasskeyKit integration for smart wallet authentication. This enables users to sign transactions using passkeys (WebAuthn) instead of secret keys.

<Callout type="info">
**passkey-kit v0.12.0** — Launchtube has been replaced by the [OpenZeppelin Relayer](https://channels.openzeppelin.com). New env vars: `RELAYER_URL` + `RELAYER_API_KEY`. See [Environment Variables](#environment-variables) below.
</Callout>

> [!NOTE]
> PasskeyKit is only available in TypeScript servers. Python servers don't support passkey signing.

## What is PasskeyKit?

PasskeyKit is a Stellar smart wallet SDK that enables:
- Passkey-based authentication (biometric, security keys)
- Multi-signer support
- Policy enforcement
- Gasless transactions (with OZ Relayer fee sponsorship)

## How Passkey Signing Works

PasskeyKit uses a two-phase signing approach:

```
Phase 1: Auth Entry Signing
  - Uses WALLET_SIGNER_SECRET
  - Signs the invocation authorization
  - Sequence-independent

Phase 2: Transaction Submission
  - OZ Relayer pays transaction fees
  - Submits via ChannelsTransactionResponse
```

### signAndSendWithPasskey()

Located in `src/lib/passkey.ts`. Initialises PasskeyKit with the OZ Relayer and uses `server.send()` which handles fee sponsorship and submission:

```typescript
export async function signAndSendWithPasskey(
  xdr: string,
  walletContractId: string
): Promise<{ hash: string; status: string }> {
  // Initialize PasskeyKit with OZ Relayer
  const passkey = getPasskeyWallet(walletContractId);

  // Sign auth entries with wallet signer
  const signerSecret = process.env.WALLET_SIGNER_SECRET;
  const signerKeypair = Keypair.fromSecret(signerSecret);

  const assembled = await passkey.sign(
    xdr,  // XDR string accepted directly (avoids stellar-sdk type collision)
    { keyId: signerKeypair }
  );

  // Submit via OZ Relayer — handles fee payment and polling
  const response = await passkey.server.send(assembled);
  // response: ChannelsTransactionResponse { hash, transactionId, status }

  return {
    hash: response.hash ?? response.transactionId,
    status: response.status,
  };
}
```

### getPasskeyWallet() — v0.12 API

```typescript
import { PasskeyKit, PasskeyServer } from 'passkey-kit';

function getPasskeyWallet(walletContractId: string) {
  const server = new PasskeyServer({
    rpcUrl: process.env.RPC_URL!,
    walletWasmHash: process.env.WALLET_WASM_HASH!,
    relayerUrl: process.env.RELAYER_URL!,      // OZ Relayer URL
    relayerApiKey: process.env.RELAYER_API_KEY!, // OZ Relayer API key
  });

  return new PasskeyKit({
    rpcUrl: process.env.RPC_URL!,
    networkPassphrase: process.env.NETWORK_PASSPHRASE!,
    walletWasmHash: process.env.WALLET_WASM_HASH!,
    server,
  });
}
```

## Signer vs Fee Payer

| Role | Environment Variable | Purpose |
|------|---------------------|---------|
| **Wallet Signer** | `WALLET_SIGNER_SECRET` | Signs auth entries (authorizes operations) |
| **Fee Sponsor** | OZ Relayer (`RELAYER_API_KEY`) | Pays transaction fees via the relayer |

The OZ Relayer handles fee payment, so users need no XLM in their wallet.

## Troubleshooting

### "WALLET_WASM_HASH not set"

Set the environment variable. Use the pre-deployed testnet hash:

```bash
export WALLET_WASM_HASH=b3eba29c256a2458faff84fc872226d29df37ad05a6a1c0efe2a95817d6765ec
```

### "Wallet contract not found"

The wallet contract may not be deployed. Run:

```bash
pnpm deploy-passkey <wasm-hash>
```

### "Auth entry signing failed"

Check that `WALLET_SIGNER_SECRET` is a valid Stellar secret key (starts with 'S').

### "Relayer API key expired or invalid"

Get a new free testnet API key at [https://channels.openzeppelin.com/testnet/gen](https://channels.openzeppelin.com/testnet/gen) and update `RELAYER_API_KEY`.

### "Fee payer account not found"

With the OZ Relayer, users do not need a funded account — the relayer sponsors fees. If you see this error, ensure `RELAYER_URL` and `RELAYER_API_KEY` are set correctly.
